# 패널 30인 회의 — 블로그·공지·관리자 배치

> 안건: 공식 블로그 노출, 소식 채널, 공문/공지 게시판, 관리자 기능

---

## 역할 분리 (합의)

| 채널 | 성격 | 배치 | 운영 |
|------|------|------|------|
| **공식 블로그** (네이버) | 일상·활동·스토리 소식 | 홈 **하단**(문의 직전) + 푸터 링크 | 기존 블로그 계속 사용 |
| **공지·공문 게시판** | 공식 안내, 공문, 입찰/운영 공지 | GNB **공지** `/notice` | 사이트 관리자 작성 |
| **관리자** | 글 등록·수정·삭제 | `/admin` (메뉴 비노출) | ID/PW 로그인 |

**소식은 블로그에서**, **공식 공지·공문은 사이트 게시판에서** — 혼동 방지.

---

## 배치 상세

1. **홈 하단** — FAQ와 문의 사이: 「공식 블로그 · 소식」 섹션  
   - CTA: 네이버 블로그 바로가기 / 인스타그램  
2. **GNB** — `공지` 추가 (소개·서비스·지점·성과·**공지**·문의)  
3. **푸터** — 블로그·공지 텍스트 링크  
4. **관리자** — URL 직접 접근만 (`/admin`), 공개 메뉴에 넣지 않음  

---

## 관리자 보안 (패널 주의)

| 항목 | 결정 |
|------|------|
| 1차 | 사이트 내 관리자 로그인 + 브라우저 저장(공지) |
| 한계 | 정적 호스팅이라 비밀번호가 프론트에 포함될 수 있음 → **운영용 임시** |
| 후속 | DB(Supabase 등) 연동 시 서버 인증으로 교체 권고 (P19·P29) |
